# 仕様書

この文書は `discordbot-auto-reaction` の実装仕様を、コード上の挙動に基づいて整理したものです。

## 1. 目的

Discord サーバー内のチャンネルごとに自動リアクション設定を保存し、以後そのチャンネルに投稿されたメッセージへ設定済み絵文字を自動付与する。

付随機能として、既存メッセージへのリアクション補完、自分のメッセージに付いたリアクション内訳の DM 確認を提供する。

## 2. 対象範囲

### 対象

- チャンネル単位の自動リアクション設定
- Unicode 絵文字と Discord カスタム絵文字の抽出
- 投稿メッセージへの自動リアクション付与
- チャンネル履歴全体へのリアクション補完
- 単一メッセージへのリアクション補完
- 自分のメッセージに付いたリアクションのユーザー内訳確認
- MongoDB による設定永続化
- 起動時の設定キャッシュ復元

### 対象外

- リアクションロール
- 投票の締切・集計・結果確定
- サーバー単位、カテゴリ単位の一括設定
- メッセージ本文に応じた条件分岐
- Web ダッシュボード
- 複数 Bot プロセス間のキャッシュ同期

## 4. 起動仕様

`main.py` の `MyBot.setup_hook()` で次の初期化を行う。

1. `extensions.auto_reaction` をロードする
2. `extensions.reaction_listup` をロードする
3. `bot.tree.sync()` でアプリケーションコマンドを同期する
4. `cache_emojis()` で MongoDB の設定をメモリキャッシュへ読み込む

Bot は `discord.Intents.all()` を使用する。Bot メンションによる設定コマンドはメッセージ本文を読むため、Discord Developer Portal で Message Content Intent を有効にする必要がある。

## 6. Extension 仕様

### 6.1 `extensions.auto_reaction`

自動リアクション、設定確認、補完を担当する。

| 種別 | 名称 | 役割 |
| --- | --- | --- |
| Event Listener | `on_message` | 設定変更、設定解除、自動リアクション付与 |
| Slash Command | `/リアクション設定確認` | 現在チャンネルの設定確認 |
| Slash Command | `/リアクション補完` | 現在チャンネルの全履歴へ補完 |
| Message Context Menu | `リアクション補完` | 選択メッセージへ補完 |

### 6.2 `extensions.reaction_listup`

リアクション内訳確認を担当する。

| 種別 | 名称 | 役割 |
| --- | --- | --- |
| Message Context Menu | `リアクション確認` | 自分のメッセージに付いたリアクション内訳を DM 送信 |

## 8. 自動リアクション設定

### 8.1 設定単位

設定は Discord チャンネル ID 単位で保存する。

### 8.2 設定判定

`on_message` で次の条件を満たす場合、設定操作として扱う。

1. メッセージ本文が空でない
2. `message.content.split()[0]` が Bot のメンションと一致する
3. 投稿者が Bot ではない
4. 投稿者が `manage_channels` 権限を持つ
5. メッセージから 1 つ以上の絵文字を抽出できる

### 8.3 設定処理

1. `extract_emojis()` で絵文字を抽出する
2. `upsert_emojis(channel_id, emojis)` を実行する
3. MongoDB の `auto_reaction.emojis` を upsert する
4. `emojis_cache[channel_id]` を更新する
5. `自動リアクション設定を追加しました` と返信する
6. 設定絵文字を Embed で表示する
7. 返信は `delete_after=10` により 10 秒後に削除される

### 8.4 解除判定

Bot メンションのみの投稿で、かつ投稿者が `manage_channels` 権限を持つ場合、設定解除として扱う。

### 8.5 解除処理

1. `delete_emojis(channel_id)` を実行する
2. MongoDB の該当ドキュメントを削除する
3. 削除件数が 1 件以上の場合、メモリキャッシュからも削除する
4. `自動リアクション設定を解除しました` と返信する
5. 返信は 10 秒後に削除される

## 9. 自動リアクション付与

設定操作・解除操作に該当しなかったメッセージについて、現在チャンネルの設定を参照し、設定済み絵文字を順番に `message.add_reaction(emoji)` で付与する。

設定が存在しない場合は何もしない。

リアクション付与時に `discord.errors.HTTPException` が発生した場合、その絵文字はスキップする。ユーザー通知は行わない。

## 10. 設定確認

`/リアクション設定確認` は、現在チャンネルの設定を取得して表示する。

- 設定あり: `設定した絵文字` という Embed で表示する
- 設定なし: `リアクション設定がありません` を返す

## 11. チャンネル履歴への補完

`/リアクション補完` は、現在チャンネルの全履歴を対象にする。

処理手順は次のとおり。

1. Interaction を ephemeral で defer する
2. 現在チャンネルの設定を取得する
3. 設定がなければ `リアクション設定がありません` を返して終了する
4. 実行開始メッセージを ephemeral で送信する
5. `interaction.channel.history(limit=None)` で全履歴を走査する
6. 各メッセージに設定済み絵文字を付与する
7. 完了メッセージを ephemeral で送信する

API リクエスト数は概ね `対象メッセージ数 × 設定絵文字数` に比例する。

## 12. 単一メッセージへの補完

右クリックメニュー `リアクション補完` は、選択した 1 件のメッセージに対して、現在チャンネルの設定済み絵文字を付与する。

設定が存在しない場合は `リアクション設定がありません` を返す。

## 13. リアクション内訳確認

右クリックメニュー `リアクション確認` は、対象メッセージが実行者本人の投稿である場合のみ実行できる。

対象メッセージの投稿者が実行者と異なる場合は、`自分のメッセージのみ確認することができます` を ephemeral で返す。

処理手順は次のとおり。

1. 対象メッセージの投稿者が実行者本人か確認する
2. `DMにリアクションの内訳を送信します` を ephemeral で返す
3. `message.reactions` を走査する
4. Bot ユーザーを除外してリアクションユーザーを取得する
5. `user.name` で昇順ソートする
6. リアクションごとに Embed を作成する
7. 実行者の DM へ送信する

10 人以下のリアクションは 1 つの Embed にまとめる。11 人以上の場合は、1 Embed あたり最大 25 人、1 回の DM 送信あたり最大 4 Embed に分割して送信する。

## 14. 絵文字抽出仕様

`utils.__init__.py` で次の関数を提供する。

| 関数 | 内容 |
| --- | --- |
| `extract_unicode_emojis(text)` | Unicode 絵文字のみ抽出 |
| `extract_custom_emojis(text)` | Discord カスタム絵文字のみ抽出 |
| `extract_emojis(text)` | Unicode 絵文字と Discord カスタム絵文字を抽出 |
| `convert_emoji(emoji, guild)` | 数値 ID の場合は Guild から絵文字を fetch して文字列表現に変換 |

`extract_emojis()` は `emoji.EMOJI_DATA` と `<:name:id>` / `<a:name:id>` 形式を対象にする。

## 16. キャッシュ仕様

`caches.py` の `emojis_cache` をメモリキャッシュとして使用する。

推奨定義は次のとおり。

```python
emojis_cache: dict[int, list[str]] = {}
```

起動時に MongoDB から全件読み込み、`channel_id` をキーとして展開する。設定追加・更新・削除時は MongoDB とキャッシュの両方を更新する。

## 17. 権限仕様

### Bot に必要な権限

- チャンネルを見る
- メッセージを送信
- メッセージ履歴を読む
- リアクションを追加
- 外部の絵文字を使用
- 埋め込みリンク
- アプリケーションコマンドを使用

### ユーザーに必要な権限

| 操作 | 条件 |
| --- | --- |
| 設定・解除 | チャンネル管理権限 |
| 設定確認 | 管理者 |
| チャンネル補完 | 管理者 |
| 単一メッセージ補完 | 管理者 |
| リアクション内訳確認 | 対象メッセージの投稿者本人 |

## 18. 応答文言

| 状況 | 文言 |
| --- | --- |
| 設定成功 | `自動リアクション設定を追加しました` |
| 解除成功 | `自動リアクション設定を解除しました` |
| 設定なし | `リアクション設定がありません` |
| チャンネル補完開始 | `チャンネル内のすべてのメッセージにリアクションを付けます` |
| チャンネル補完完了 | `チャンネル内のすべてのメッセージにリアクションを付けました` |
| 単一補完完了 | `メッセージ {message.jump_url} にリアクションを付けました` |
| 他人のメッセージ確認不可 | `自分のメッセージのみ確認することができます` |
| DM 送信案内 | `DMにリアクションの内訳を送信します` |

## 19. 例外処理

- コマンド・イベントには Daug の `excepter` / `dpylogger` が付与されている
- `message.add_reaction()` の `discord.errors.HTTPException` は握りつぶす
- リアクション付与失敗の件数・理由はユーザーへ通知しない
- DM 送信失敗の個別メッセージは現行実装では定義されていない

## 20. 制約

- 設定はチャンネル単位のみ
- 設定絵文字数の上限は実装上未定義
- 履歴補完の対象件数上限は実装上未定義
- 履歴補完のキャンセル機能はない
- 履歴補完の進捗表示はない
- 複数プロセス起動時のキャッシュ同期はない
- DB を直接編集しても稼働中 Bot には即時反映されない
- `constants.py` と `caches.py` はリポジトリ外で用意する必要がある

## 21. 推奨改善

- `constants.py.example` と `caches.py` を追加する
- Intents を必要最小限に絞る
- MongoDB の `channel_id` に unique index を設定する
- 履歴補完に確認 UI、件数上限、進捗表示を追加する
- リアクション付与失敗件数を集計して通知する
- DM 送信失敗時のユーザー通知を追加する
- 複数プロセス運用時のキャッシュ更新戦略を定義する
